## Why v5 spec changed (delta+velmode ‚Üí sub+abs+velmode)

User had v5 = `v5-lora-r32-full-delta-velmode` in the table. Two reasons to update:
1. **Encoding**: re-reading the code (`handler.py:65-80` + `X-VLA/datasets/utils.py:99-104` + `eval.py:147-148`), delta training learns command-relative deltas but inference adds live state. With controller lag, inferred target ‚âà lagged_state + small_delta. Pairing delta with velocity (`(target ‚àí live)/dt`) compounds (small_delta ‚Üí small_twist ‚Üí freeze). Absolute is the clean pair for velocity.
2. **Data**: the colleague's freshest re-upload is `siyulw2025/aic-data-sub-v2` (103K, P3-trimmed), not the full 270K. The full version doesn't exist on HF yet.

So v5 ‚Üí `v5-lora-r32-sub-abs-velmode`. This is essentially "v3 done right": same sub data + absolute encoding as v3, but with proper VLM LR (`learning_coef=1.0`, NOT v3's 0.1) + LoRA rank 32 + velocity control.
